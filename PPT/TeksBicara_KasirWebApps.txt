  SLIDE 01  |  COVER — Kasir Web Apps
======================================================================

Assalamualaikum warahmatullahi wabarakatuh.

Selamat pagi, Bapak/Ibu dosen dan rekan-rekan semua. Perkenalkan, kami dari Kelompok 5 akan mempresentasikan proyek akhir kami yang berjudul Kasir Web Apps.

Kasir Web Apps adalah aplikasi kasir restoran berbasis web yang sepenuhnya berjalan di sisi klien — tidak membutuhkan server backend sama sekali. Semua proses, mulai dari login, manajemen menu, transaksi, hingga cetak struk PDF, semuanya terjadi langsung di browser pengguna.

Aplikasi ini dibangun dengan tiga teknologi utama: HTML5 untuk struktur halaman, Vanilla JavaScript ES6 plus untuk seluruh logika program, dan Bootstrap 5 untuk tampilan yang modern dan responsif.

Ada empat pilar fitur utama: sistem autentikasi berbasis localStorage dan Cookie, manajemen menu produk dari file JSON, keranjang belanja dengan kalkulasi pajak otomatis, dan cetak struk PDF langsung dari browser. Silakan ikuti presentasi kami.


======================================================================
  SLIDE 02  |  OVERVIEW FITUR APLIKASI
======================================================================

Baik, sekarang kita masuk ke slide kedua yang menampilkan overview enam fitur utama aplikasi.

Fitur pertama adalah Login dan Register. Pengguna harus melakukan autentikasi sebelum bisa mengakses halaman manapun di aplikasi ini. Status login disimpan secara bersamaan di localStorage dan Cookie browser.

Fitur kedua adalah Dashboard Menu. Di sini semua produk ditampilkan dalam bentuk kartu yang dikelompokkan per kategori, seperti Makanan, Minuman, dan Makanan Ringan. Data produk diambil dari file menu.json.

Fitur ketiga adalah Keranjang Belanja. Pengguna bisa menambah produk, mengubah jumlah, dan menghapus item. Kalkulasi total dilakukan secara otomatis dengan diskon 10 persen ditambah PPN 10 persen.

Fitur keempat adalah Struk PDF. Setelah transaksi selesai, sistem dapat menghasilkan file PDF struk langsung di browser menggunakan library jsPDF, tanpa perlu kirim data ke server.

Fitur kelima adalah Tambah Produk. Admin dapat menambahkan produk baru melalui form khusus. Produk tersebut langsung tersedia di menu utama kasir.

Fitur keenam adalah Auth Guard. Mekanisme keamanan yang memastikan setiap halaman hanya bisa diakses oleh pengguna yang sudah login. Kalau belum login, otomatis diarahkan ke halaman login.


======================================================================
  SLIDE 03  |  STRUKTUR FILE PROYEK
======================================================================

Slide ini menjelaskan struktur file proyek. Kami membaginya menjadi tiga kelompok besar.

Kelompok pertama adalah HTML Pages. Ada tiga halaman utama: index.html sebagai halaman kasir utama tempat transaksi terjadi, login.html untuk proses autentikasi pengguna, dan addproduct.html khusus untuk admin menambah produk baru.

Kelompok kedua adalah JavaScript Files yang jumlahnya ada lima. cookieUtils.js harus selalu di-load pertama karena berisi fungsi dasar get, set, dan delete cookie yang dibutuhkan file lainnya. auth.js berisi fungsi isLoggedIn, setLoggedIn, dan clearLoggedIn yang mengelola state login. loginStyle.js mengatur semua animasi, validasi form, dan logika khusus halaman login. main.js adalah file terbesar yang mengatur seluruh alur kasir mulai dari load menu, cart, kalkulasi, hingga generate PDF. Dan addProduct.js yang baru kami tambahkan untuk mengatur form dan tabel produk custom.

Kelompok ketiga adalah Data dan Assets. menu.json menyimpan data produk default yang di-fetch saat aplikasi pertama dibuka. localStorage dipakai untuk menyimpan cart, status login, dan produk custom. Cookie menyimpan token autentikasi sesi selama satu hari. Dan css/style.css mengatur tampilan visual keseluruhan aplikasi.


======================================================================
  SLIDE 04  |  ALUR LOGIKA — AUTENTIKASI
======================================================================

Sekarang kita lihat alur logika autentikasi secara detail.

Pertama, saat pengguna membuka login.html, hal pertama yang terjadi adalah pengecekan fungsi isLoggedIn. Fungsi ini mengecek dua kondisi sekaligus: apakah localStorage menyimpan nilai kasir_logged_in sama dengan true, DAN apakah Cookie kasir_auth juga bernilai true. Kedua syarat ini harus terpenuhi bersamaan.

Kalau pengguna sudah login sebelumnya, sistem langsung redirect ke index.html tanpa perlu login lagi.

Kalau belum login, pengguna mengisi form email dan password. Validasi dilakukan secara lokal — ini adalah simulasi autentikasi untuk keperluan demo, bukan autentikasi ke server nyata. Email valid adalah zhaenx at yeswehack.ninja dengan password zh43nx.

Jika kredensial benar, fungsi setLoggedIn dipanggil. Fungsi ini menyimpan tiga item ke localStorage yaitu status login, email pengguna, dan timestamp login. Sekaligus menyimpan dua Cookie: kasir_auth dan kasir_email, keduanya berlaku selama satu hari.

Jika salah satu sisi dihapus, misalnya localStorage dibersihkan tapi Cookie masih ada, fungsi isLoggedIn tetap mengembalikan false dan pengguna harus login ulang. Ini adalah lapisan keamanan tambahan agar tidak mudah dimanipulasi lewat DevTools.


======================================================================
  SLIDE 05  |  ALUR LOGIKA — KASIR & KERANJANG
======================================================================

Mari kita bahas alur logika bagian kasir dan keranjang belanja.

Langkah pertama adalah loadMenu. Fungsi ini mengambil data dari file menu.json menggunakan fetch API, kemudian menggabungkannya dengan produk custom yang tersimpan di localStorage melalui fungsi mergeCustomProducts. Hasilnya disimpan di variabel global menuData.

Langkah kedua adalah renderMenu yang mengiterasi menuData dan menghasilkan kartu produk HTML untuk setiap item, dikelompokkan berdasarkan kategori dengan header dan ikon masing-masing.

Langkah ketiga, saat pengguna mengklik tombol Add to Cart, fungsi addToCart dipanggil. Item disimpan ke objek cart di memori sekaligus dipersiskan ke localStorage agar tidak hilang saat halaman di-refresh.

Langkah keempat, renderCart memperbarui tampilan keranjang di panel offcanvas dengan qty, subtotal per item, dan tombol aksi.

Langkah kelima adalah calculateTotals yang menjalankan rumus kalkulasi. Subtotal dihitung dari jumlah price dikali qty semua item. Kemudian dipotong diskon 10 persen menghasilkan DPP atau Dasar Pengenaan Pajak. DPP kemudian dikenai PPN 10 persen. Total akhir adalah DPP ditambah PPN. Dan kembalian adalah uang bayar dikurangi total.


======================================================================
  SLIDE 06  |  ALUR LOGIKA — TAMBAH PRODUK
======================================================================

Slide ini menjelaskan alur logika fitur tambah produk yang ada di halaman addproduct.html.

Langkah pertama, begitu halaman dibuka, auth guard langsung dijalankan. Jika pengguna belum login, langsung diarahkan ke login.html. Ini mencegah akses tidak sah ke halaman admin.

Langkah kedua, admin mengisi form yang terdiri dari nama produk, harga, kategori, dan URL gambar yang bersifat opsional.

Langkah ketiga adalah validasi. Nama produk minimal dua karakter, harga harus lebih dari nol, dan kategori wajib dipilih. Jika ada yang tidak memenuhi syarat, muncul toast notifikasi peringatan.

Langkah keempat adalah pengecekan duplikat. Sistem memeriksa apakah produk dengan nama yang sama di kategori yang sama sudah ada di localStorage. Jika ya, proses dihentikan dengan notifikasi.

Langkah kelima, fungsi generateId membuat ID unik dengan format custom underscore timestamp underscore angka random. Format ini memastikan tidak ada bentrokan dengan ID produk default dari menu.json.

Langkah keenam dan ketujuh, produk disimpan ke localStorage dengan key kasir_custom_products, tabel produk diperbarui, dan toast notifikasi sukses ditampilkan di pojok kanan bawah layar.

======================================================================
  SLIDE 10  |  TEKNOLOGI & LIBRARY
======================================================================

Slide ini merangkum semua teknologi dan library yang kami gunakan dalam proyek ini.

Pertama, HTML5 dan CSS3 sebagai fondasi struktur dan tampilan halaman. CSS custom dipakai untuk animasi login, kartu produk, dan tema warna aplikasi.

Kedua, Vanilla JavaScript ES6 plus sebagai inti logika. Kami memilih Vanilla JS tanpa framework untuk membuktikan bahwa fitur-fitur kompleks bisa dibangun hanya dengan JavaScript murni.

Ketiga, Bootstrap 5.3 untuk komponen UI siap pakai seperti modal, offcanvas, navbar, dan grid system. Ini mempercepat pengembangan tampilan yang responsif.

Keempat, Bootstrap Icons yang menyediakan lebih dari 1300 ikon SVG untuk memperindah antarmuka.

Kelima, jsPDF adalah library JavaScript yang paling krusial. Library ini memungkinkan pembuatan dokumen PDF langsung di browser tanpa butuh server — inilah yang membuat fitur struk kami bisa berjalan offline.

Keenam, ScrollReveal.js untuk animasi elemen saat di-scroll, memberikan kesan presentasi yang lebih dinamis.

Ketujuh dan kedelapan, localStorage dan Cookie API dari browser. Keduanya dimanfaatkan sebagai pengganti database dan session server. Ini yang memungkinkan seluruh aplikasi berjalan tanpa backend.

  SLIDE 11  |  KESIMPULAN
======================================================================

Sebagai penutup presentasi kami.

Proyek Kasir Web Apps ini berhasil membuktikan bahwa sebuah aplikasi kasir yang lengkap dan fungsional dapat dibangun sepenuhnya berbasis browser tanpa memerlukan server backend sama sekali.

Poin pertama: aplikasi ini adalah full-stack di browser. Semua lapisan — data, logika, dan tampilan — berjalan di satu tempat yaitu browser pengguna.

Poin kedua: sistem autentikasi ganda dengan localStorage dan Cookie memberikan lapisan keamanan tambahan yang lebih baik dibanding hanya mengandalkan satu penyimpanan.

Poin ketiga: keranjang belanja persisten yang tidak hilang saat refresh, dengan kalkulasi pajak otomatis sesuai aturan PPN.

Poin keempat: fitur struk PDF langsung dari browser adalah nilai lebih yang membuat aplikasi ini benar-benar siap pakai untuk operasional restoran sederhana.

Poin kelima: admin bisa menambah produk baru kapan saja tanpa perlu mengubah kode.

Poin keenam: aplikasi ini zero-dependency install, cukup buka file index.html di browser manapun, langsung bisa digunakan.

======================================================================
  SLIDE 12  |  CARA MENJALANKAN DI NetBeans IDE
======================================================================

Slide ini adalah panduan praktis untuk menjalankan proyek di NetBeans IDE.

Langkah pertama adalah instalasi. Download NetBeans versi terbaru dari netbeans.apache.org. Setelah terinstall, buka menu Tools lalu Plugins, cari dan install plugin PHP dan HTML5, kemudian restart NetBeans. Pastikan juga JDK versi 11 ke atas sudah terinstall di komputer karena NetBeans membutuhkannya.

Langkah kedua, buka proyek dengan File lalu Open Project, atau shortcut Ctrl Shift O. Arahkan ke folder proyek Kasir Web Apps. NetBeans akan otomatis mendeteksinya sebagai project HTML atau PHP. Pastikan struktur folder terlihat lengkap: ada file index.html, folder js berisi semua JavaScript, folder css, dan folder data yang berisi menu.json.

Langkah ketiga adalah konfigurasi server. Klik kanan nama project, pilih Properties, buka tab Run. Pilih Built-in Web Server yang berjalan di port 8080 secara default. Kalau ingin menggunakan XAMPP, salin seluruh folder proyek ke dalam folder htdocs di instalasi XAMPP, kemudian akses lewat localhost garis miring kasir. Ini poin yang sangat penting: proyek ini WAJIB dijalankan melalui server, tidak boleh dibuka langsung dengan double-click file HTML karena browser modern memblokir fetch request ke file lokal karena kebijakan CORS.

Langkah keempat dan terakhir, jalankan proyek dengan klik tombol Run berwarna hijau di toolbar NetBeans atau cukup tekan F6. Browser akan terbuka secara otomatis menuju localhost port 8080. Halaman login akan muncul. Masukkan email zhaenx@yeswehack.ninja dan password zh43nx untuk masuk ke dashboard kasir.

Jika browser tidak terbuka otomatis, buka manual dan ketik localhost:8080/index.html di address bar.
